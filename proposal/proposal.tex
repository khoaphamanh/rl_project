\section{Motivation}

Standard RL algorithms like PPO, DQN assume the Markov property: the optimal action depends only on the current state. However, many environments are partially observable, means, the current observation alone is insufficient to act optimally. State aliasing occurs when different true states look identical to the agent, yet require different actions.

\textbf{Example:} In \texttt{MiniGrid-Memory}, the agent must remember an object it saw earlier, then select the matching object later. The choice point produces identical observations regardless of what was seen before, memoryless PPO and DQN can only guess to solve the task, find the same object as seen before.

We address this by introducing recurrent agents that maintain a hidden state encoding observation history. Instead of deciding actions from observation alone, they condition on both past hidden state and current observation. 

\textbf{Research Question:} Does equipping PPO and DQN with recurrent memory recover performance in partially observable environments, and how much memory capacity and history length are actually needed?

We benchmark recurrent agents against baselines on memory-critical environments with ablations controlling observability level, memory capacity, and BPTT truncation horizon.

\section{Related Topics}

In the reinforcement learning course, algorithms such as DQN and PPO have been introduced and applied to simple environments like LunarLander and CartPole. These algorithms operate under the MDP assumption.

In Deep Learning, recurrent neural network layers such as LSTM and GRU possess the ability to maintain and integrate information from previous time steps, preserving temporal dependencies and event ordering. This property has been successfully applied in domains such as natural language processing.

The seminal paper "Deep Recurrent Q-Learning for Partially Observable MDPs" by Hausknecht et al. first combined DQN with LSTM in the Atari 2600 Game environment under the POMDP assumption \cite{drqn}. More recently, "Generalization, Mayhem and Limits in Recurrent Proximal Policy Optimization" by Pleines et al. (2022) introduced RPPO, which integrates PPO with LSTM/GRU architectures for partially observable environments \cite{rppo}.

\section{Experiments}
\subsection{Enviroment}
We adopt the Memory MiniGrid environment, where the agent begins in a left room with a 7×7 observation field. The task is to identify an object (green key) in the initial room, then proceed to a fork on the right to select the matching object. This instantiates a POMDP: task completion depends on the agent remembering the object's features across time steps. Reward is sparse, provided only upon task success. As shown in Fig. \ref{fig:memory_grid}, we start with \texttt{MiniGrid-MemoryS7-v0} and plan to scale to \texttt{MiniGrid-MemoryS9-v0} or \texttt{MiniGrid-MemoryS11-v0} if preliminary results are favorable.

\subsection{RPPO}

We employ a CNN1D encoder combined with an LSTM layer, shared between the actor and critic networks. Both networks use two separate heads for policy and value outputs, following the architecture proposed by Pleines et al.

\textbf{Sequence Padding: }
Since episodes have variable lengths until termination, we pad all sequences to a uniform length to satisfy LSTM's input requirements. Embeddings corresponding to padded positions are masked and excluded from loss computation across all three PPO losses (policy loss, value loss, and entropy bonus).

\textbf{Masking Example:}
Given batch size 2:
\begin{align}
\text{Episode 1:} \quad &[t_1, t_2, t_3, \text{PAD}, \text{PAD}] \\
\text{Episode 2:} \quad &[t_1, t_2, t_3, t_4, t_5]
\end{align}

After LSTM encoding and masking:
\begin{align}
\text{Episode 1:} \quad &[0.8, 0.6, 0.4, 0.0, 0.0] \\
\text{Episode 2:} \quad &[0.5, 0.3, 0.7, 0.4, 0.6]
\end{align}

The effective batch contains 8 steps (3 from Episode 1, 5 from Episode 2). The remaining PPO loss terms remain unchanged.

\subsection{Hyperparameter Optimization}
We conduct hyperparameter tuning for the LSTM architecture, focusing on input size, hidden size, and sequence length to analyze their impact on training dynamics. For PPO, we tune only the learning rate. The objective function is the total episode reward.

\subsection{Benchmarking}

We benchmark RPPO against PPO and DQN baselines under the POMDP assumption on Memory MiniGrid. We expect RPPO to substantially outperform memoryless PPO and DQN. All methods are subjected to hyperparameter optimization and evaluated across multiple random seeds.
